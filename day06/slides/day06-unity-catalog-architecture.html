What is Unity Catalog?

Unity Catalog is Databricks' unified governance solution for all data and AI assets, providing centralized access control, auditing, lineage, and discovery.

  • Centralized metadata and governance
  • Fine-grained access control (tables, columns, rows)
  • Automatic data lineage tracking
  • Cross-workspace data sharing
  • Works with all Databricks workloads

Three-Level Namespace

CATALOG
↓
SCHEMA (Database)
↓
TABLE / VIEW / FUNCTION
-- Fully qualified name
SELECT * FROM catalog_name.schema_name.table_name;

The Metastore

Metastore is the top-level container for all Unity Catalog objects. One metastore per region, shared across workspaces.

  • Contains catalogs, schemas, tables
  • Stores access control policies
  • Tracks lineage information
  • Region-specific deployment
  • Assigned to workspaces in same region

Storage Credentials & External Locations

Storage Credentials
  • Reference to cloud IAM
  • AWS: IAM Role
  • Azure: Service Principal
  • GCP: Service Account
External Locations
  • Path in cloud storage
  • Uses storage credential
  • For external tables
  • Scoped access control

Managed vs External Tables

Aspect Managed Table External Table
Data Location Metastore storage Your specified path
Data Lifecycle Unity Catalog manages You manage
DROP TABLE Deletes data Keeps data
Use Case New data, full governance Existing data, external access

Centralized Governance Model

  • Unified permissions – GRANT/REVOKE SQL syntax
  • Inheritance – Permissions cascade down hierarchy
  • Data lineage – Automatic tracking of data flow
  • Audit logging – All access is logged
  • Data discovery – Search and browse all assets

Unity Catalog Admin Tasks

Metastore Admin

  • Create catalogs
  • Manage storage credentials
  • Configure external locations
  • Assign metastore to workspaces

Catalog Admin

  • Create schemas
  • Grant catalog access
  • Manage catalog permissions

Data Owner

  • Create tables/views
  • Grant table access
  • Manage object permissions
