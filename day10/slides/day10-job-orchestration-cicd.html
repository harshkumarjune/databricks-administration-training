What are Databricks Jobs?

Databricks Jobs are the mechanism for running notebooks, Python scripts, JARs, and other code on a scheduled or triggered basis.

  • Production workload execution
  • Scheduled or triggered runs
  • Automatic cluster management
  • Built-in monitoring and alerts
  • Retry and failure handling

Job Task Types

Notebook

  • Run Databricks notebooks
  • Pass parameters
  • Support all languages

Python Script

  • Run .py files from repos
  • Command-line args
  • Wheel packages

JAR/Spark Submit

  • Java/Scala applications
  • Main class execution
  • Library dependencies

SQL

  • Run SQL queries
  • SQL warehouse or cluster
  • Dashboard refresh

dbt

  • Run dbt projects
  • Transformation pipelines
  • Native integration

Delta Live Tables

  • Run DLT pipelines
  • Declarative ETL
  • Quality enforcement

Schedule Configuration

# Cron expression format
# ┌───────────── minute (0-59)
# │ ┌───────────── hour (0-23)
# │ │ ┌───────────── day of month (1-31)
# │ │ │ ┌───────────── month (1-12)
# │ │ │ │ ┌───────────── day of week (0-6, Sun=0)
# * * * * *

Examples:
"0 8 * * *" # Daily at 8:00 AM
"0 0 * * 0" # Weekly on Sunday midnight
"0 */2 * * *" # Every 2 hours
"0 9 1 * *" # Monthly on 1st at 9 AM

File Arrival Trigger

Automatically trigger jobs when new files arrive in cloud storage locations.

{
  "file_arrival": {
    "url": "s3://my-bucket/incoming/",
    "min_time_between_triggers_seconds": 60,
    "wait_after_last_change_seconds": 30
  }
}
  • S3, ADLS, GCS supported
  • Configurable polling interval
  • Debounce for batch arrivals

Run As Configuration

Run As determines which identity executes the job, affecting data access permissions.

Service Principal
  • Recommended for production

  • No user dependency
User Identity
  • Runs as job owner

Databricks Asset Bundles (DABs)

  • Infrastructure as Code for Databricks
  • Define jobs, pipelines, clusters in YAML
  • Environment-specific configurations
  • Version control friendly
  • CLI-based deployment
# databricks.yml
bundle:
  name: my-etl-pipeline

resources:
  jobs:
    daily_etl:
      name: "Daily ETL Job"
      schedule:
        quartz_cron_expression: "0 0 8 * * ?"

DABs Environments

# databricks.yml
targets:
  dev:
    workspace:
      host: https://dev.cloud.databricks.com
    variables:
      catalog: dev_catalog

  prod:
    workspace:
      host: https://prod.cloud.databricks.com
    variables:
      catalog: prod_catalog
databricks bundle deploy --target dev
databricks bundle deploy --target prod

GitHub Actions Example

name: Deploy to Databricks
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - uses: databricks/setup-cli@main
    - run: databricks bundle deploy --target prod
      env:
        DATABRICKS_HOST: ${{ secrets.DB_HOST }}
        DATABRICKS_TOKEN: ${{ secrets.DB_TOKEN }}
